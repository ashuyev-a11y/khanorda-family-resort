# Khan Orda Family Resort — MVP-лендинг с калькулятором

Mobile-first витрина семейного Halal Friendly курорта **Khan Orda** на берегу реки Чаган (20 минут от Уральска) с фронтовым калькулятором стоимости. Всё считается на клиенте — **без бэкенда, без базы данных, ничего никуда не отправляется**.

## Стек
- **Next.js 15** (App Router) + **TypeScript** + **Tailwind CSS**
- Шрифты через **next/font**: Prata / Manrope / IBM Plex Mono (кириллица + казахские символы)
- Изображения через **next/image** (WebP/AVIF), ленивая галерея
- **RU/KZ** переключатель языков (каркас i18n)
- Иконки — **lucide-react**

## 💰 Цены — один файл: `config/pricing.ts`
**Все цены и правила меняются ТОЛЬКО здесь.** Ничего не нужно трогать в компонентах.

В `config/pricing.ts` задаются:
- **Домики** (`UNITS`): имя, вместимость, цена будни (`weekdayPrice`), выходные (`weekendPrice`), праздники (`holidayPrice`), фото, особенности.
- **Доп. услуги** (`ADDONS`): название, цена, режим `once` (за всё бронирование) или `per_night` (за ночь).
- **Выходные ночи** (`WEEKEND_NIGHTS`) — по умолчанию ночи пт/сб дороже.
- **Праздники** (`HOLIDAYS`) — список дат, которые считаются по праздничной цене.
- **Настройки** (`BOOKING`): процент предоплаты, домик по умолчанию.

> ⚠️ Сейчас в конфиге стоят **плейсхолдеры** (3 домика и услуги баня/чан/завтрак/трансфер/барбекю). Замените цифры и названия на реальные.

## Как считает калькулятор
`(цена ночи × число ночей, с учётом будни/выходные/праздники) + доп. услуги`
- Пересчёт **мгновенный** при любом изменении (без кнопки «посчитать»).
- Видна разбивка: проживание (сколько будних/выходных/праздничных ночей) + услуги = итого + предоплата.
- Если даты не выбраны/некорректны — показывается подсказка, а не пустой экран.
- Кнопка «Забронировать» **ничего не отправляет** — показывает сводку и опциональную ссылку в WhatsApp (гость отправляет сам).

## Компоненты (по блокам)
`Header · Hero · Units (Домики) · Amenities (Удобства) · Calculator · Gallery · Pricing (Цены) · Reviews (Отзывы) · Location (Локация) · FAQ · Footer · MobileBar`
Структура готова к тому, чтобы позже без переделки прикрутить отправку заявки / оплату / Supabase.

## Разработка
```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production
```

## Как вносить правки
1. **Цены/домики/услуги** → `config/pricing.ts` (один файл).
2. **Фото** → замените файлы в `public/img/` (те же имена) на реальные снимки.
3. **Тексты интерфейса** → `src/i18n/dictionaries.ts` (RU/KZ). Контент-списки (удобства, отзывы, FAQ) → `src/data/content.ts`.
4. Коммит и `git push` в ветку `main` → Vercel автоматически пересобирает и обновляет ту же production-ссылку.
